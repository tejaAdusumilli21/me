Topics & Problems Codey Tackles

The hard developer problems that live in Codey's domain

01

Apex Business Logic

Complex business rules that can't be solved declaratively. Trigger frameworks, service layers, and handler classes that process thousands of records without hitting governor limits.

02

Lightning Web Components (LWC)

Building dynamic, reusable UI components — custom data tables, modals, search bars, wizards — that feel native inside Salesforce but are fully custom for the business.

03

Governor Limit Violations

SOQL inside loops, DML in triggers, heap size blowouts — debugging and fixing code that works fine in dev but breaks in production with real data volumes.

04

Async Apex (Batch, Queueable, Future)

Processing large datasets, chaining jobs, scheduling nightly runs. Choosing the right async pattern for the right problem.

05

REST & SOAP API Integration

Calling external systems from Apex (callouts), or exposing Salesforce data via custom REST APIs. Authentication, error handling, retry logic.

06

Test Coverage & Quality

Writing test classes that actually test logic — not just pump coverage numbers. Mocking callouts, using Test.startTest(), covering bulk scenarios and negative paths.

07

Trigger Frameworks

Preventing trigger hell — one trigger per object, handler classes, preventing recursive firing, keeping business logic out of the trigger itself.

08

SOQL & SOSL Optimization

Selective queries, index usage, relationship queries, aggregate functions, and avoiding full table scans on large objects with millions of records.
